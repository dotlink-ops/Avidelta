# Vercel CI/CD Setup

This repository is configured for automatic deployments via Vercel.

## Setup Status

- **GitHub Repo:** `dotlink-ops/nextjs`
- **Production URL:** `https://avidelta.vercel.app`
- **Framework:** Next.js 16.0.0

## Quick Setup

1. **Import to Vercel:**
   - Visit: https://vercel.com/new
   - Import: `dotlink-ops/nextjs`
   - Framework: Auto-detected (Next.js)
   - Click "Deploy"

2. **Verify Integration:**
   - Go to: Project Settings → Git
   - Ensure "GitHub" is connected
   - Enable: "Automatically deploy when pushed to"

## Deployment Triggers

### Production (main branch)
- **Trigger:** Push to `main`
- **URL:** https://avidelta.vercel.app
- **Auto-deploy:** ✅ Enabled

### Preview (PRs)
- **Trigger:** Any PR to `main`
- **URL:** Unique URL per PR (e.g., `nextjs-git-feature-xyz.vercel.app`)
- **Auto-deploy:** ✅ Enabled
- **Comment:** Vercel bot adds deployment URL to PR

## Health Check URLs

Add these to every PR description for instant verification:

```
Health: https://[preview-url]/api/healthz
Status: https://[preview-url]/api/status
Daily Summary: https://[preview-url]/api/daily-summary
Sitemap: https://[preview-url]/sitemap.xml
```

## Environment Variables

No secrets required for basic functionality. All API routes work out-of-the-box.

If you add authenticated APIs:
1. Go to: Project Settings → Environment Variables
2. Add keys for: Production, Preview, Development
3. Redeploy to apply

## Verification Commands

```bash
# Production
curl https://avidelta.vercel.app/api/healthz | jq

# Preview (replace [url])
curl https://[preview-url]/api/healthz | jq
```
